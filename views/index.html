Inventories your servers and makes details about them searchable by people and queryable by software.

What server is xyz running on?

Above all else Onering exists to answer questions like this. Onering a service that stores information about assets in your organization, like servers and switches. It offers a sleek frontend for your users to perform searches and generate reports, as well as a comprehensive RESTful API that can be used by scripts and configuration management systems.

 


Turn Data Into Information

The modern technical infrastructure for many companies is vast — hundreds to thousands of different servers, virtual machines, and containers running numerous individual services. Turning a list of hostnames and IP addresses into usable information is a complicated process that requires institutional knowledge at all levels.

Onering offers you a single point of truth for your whole network; one place where any question can be asked (like “Which nodes are running nginx on port 443?” or “How many nodes have not run Chef in over an hour?”). By putting all the facts and details about your systems in the same place, new relationships between disparate data can be discovered and utilized to make managing your infrastructure quicker and safer.

 


Shared Configuration Metadata

Configuration management systems have defined a new way to perform work in operations, but at their heart they still rely on user-provided information about what goes where. Different configuration management systems offer different mechanisms to make shared metadata available to administrators. Onering provides a common API that can be used by any system as an external source of data. It marries the utility of a distributed "data structure server"; the expressiveness of a data query language; and the convenience of a clean, intuitive RESTful API into one tool.
